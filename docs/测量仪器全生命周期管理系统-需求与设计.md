# 测量仪器仪表全生命周期管理系统
## 需求文档与技术设计

> 版本：v1.3
> 日期：2026-05-13
> 编制：工艺质量管理科
> 开发：Hermes Agent（AI 自主开发与维护）

---

## 一、项目背景与现状

### 1.1 现状

工厂目前已有一套 MES 系统，其中包含测量管理模块，具备基础的仪器仪表台账管理功能，但存在以下问题：

- **功能不全**：仅能管理台账，无检定计划、证书管理、流程审批等功能
- **体验不佳**：操作繁琐，不能满足日常工作需要
- **无外部协作能力**：无法对接外部检测院（第三方检定机构）的检定数据
- **改造困难**：MES 供应商配合意愿不确定，扩展受限

### 1.2 核心痛点

除了台账和检定管理的基础需求外，当前工作中最大的痛点是：

**检定合同与实际执行的对账靠人工核对**，具体场景如下：

- 工厂每年与外部检测院签订**年度检定服务合同**，合同中约定了送检的设备名称、数量、单价、总金额
- 各内部部门也会有检定费用的分摊和结算
- 实际执行中，送检的设备、数量、费用**往往与合同不一致**（设备更换、数量增减、价格调整等）
- 目前需要人工逐一核对：
  - 合同约定的设备清单 ↔ 实际送检记录
  - 合同金额 ↔ 实际发生金额
  - 各部门的检定费用分摊是否准确
- **耗时长、易出错、缺乏追溯能力**

## 二、系统目标

| 目标 | 说明 |
|------|------|
| 统一台账 | 建立全厂统一的仪器仪表电子台账，全面替代 MES 测量模块 |
| 规范流程 | 新购、报废、借用、维修等流程线上化、审批化 |
| 智能检定 | 自动排期、到期预警、证书电子化管理 |
| 合同对账 | 自动比对合同内容与实际执行，一键识别差异 |
| 监督闭环 | 部门自查 → 工厂督查 → 整改跟踪，全流程电子化 |
| 数据驱动 | 统计分析仪表盘，辅助管理决策 |
| 权限分明 | 部门级权限控制，各管各的、领导能看全貌 |

## 四、功能需求详细说明

### 4.1 基础数据管理

#### 4.1.1 部门管理
- 维护组织机构树（工厂 → 车间/科室 → 班组）
- 每个部门可设置部门测量员、部门领导
- 支持部门费用中心（用于检定费用分摊）

#### 4.1.2 仪器分类
- 多级分类：如「长度类→卡尺→数显卡尺」
- 支持自定义分类属性、技术指标模板

#### 4.1.3 计量单位/精度等级
- 统一的计量单位和精度等级字典
- 用于仪器技术参数录入

#### 4.1.4 供应商管理
- 仪器生产厂家信息维护
- **检定服务机构（检测院）管理**：名称、资质证书、联系方式、合作有效期、服务范围、收费标准

#### 4.1.5 MES 数据迁移/对接
- **一次性迁移**：将 MES 中现有仪器台账数据导入新系统（仪器编号、名称、型号、部门归属等）
- **接口预留**：后续可考虑与 MES 做双向同步接口（如有需要）

### 4.2 台账管理（核心模块）

#### 4.2.1 仪器信息卡片
每台仪器建立唯一的电子档案卡片，包含：

| 分组 | 字段 | 说明 |
|------|------|------|
| **基本信息** | 仪器编号 | 系统自动生成（如 JL-2026-0001）|
| | 仪器名称 | 如：数显卡尺、万用表 |
| | 型号规格 | 如：0-150mm |
| | 出厂编号 | 厂家铭牌编号 |
| | 仪器分类 | 关联分类树 |
| | 精度/不确定度 | 技术指标 |
| | 测量范围 | 量程信息 |
| | 生产厂家 | 关联供应商 |
| | 出厂日期 | |
| | 购入日期 | |
| | 资产原值 | 金额 |
| | 管理部门 | 归属部门 |
| | 保管人/责任人 | 具体人员 |
| | 存放地点 | 精确位置 |
| | 状态 | 在用/停用/封存/报废/送检/维修 |
| | 附照片 | 仪器实物照片 |
| **检定信息** | 检定周期 | 如 12个月 |
| | 上次检定日期 | |
| | 下次检定日期 | 系统自动计算 |
| | 检定方式 | 送检/现场检定/自检 |
| | 当前证书 | 关联证书文件 |
| | 检定费用 | 最近一次检定实际费用 |
| **附件** | 合格证扫描件、使用说明书（PDF）、维修记录附件 |

#### 4.2.2 新购申请流程

```
1. 部门测量员填写新购申请单（仪器名称、规格、数量、预算、用途说明）
2. 本部门领导审核
3. 体系管理员复核（核查是否已有闲置可调配）
4. 分管领导审批
5. 采购执行（系统外完成，完成后在系统登记采购结果）
6. 到货验收登记（入台账）
```

- 审批流程支持会签（多人审批）和退回修改
- 每个环节留痕（操作人、时间、意见）
- 可查看申请进度（待审/已通过/已驳回）

#### 4.2.3 报废申请流程

```
1. 部门测量员发起报废申请（选择仪器、填写报废原因、附技术鉴定意见）
2. 本部门领导审核
3. 体系管理员复核
4. 分管领导审批
5. 批准后，仪器状态变更为"已报废"，归档保留记录
```

#### 4.2.4 借用/归还管理
- 部门间借用登记
- 超期提醒，借用记录可追溯

#### 4.2.5 调拨管理
- 仪器跨部门调拨，需双方部门领导确认

### 4.3 检定/校准管理（核心模块）

#### 4.3.1 检定计划制定
- **自动排期**：根据周期和上次检定日期，自动生成年度/月度检定计划
- **手动调整**：支持人工调整计划日期
- **计划视图**：年视图、月视图、日历视图
- **关联合同**：计划中的送检项目可关联到对应的检定合同条目

#### 4.3.2 计划执行
| 步骤 | 说明 |
|------|------|
| 到期提醒 | 提前N天自动通知（可配置：提前30天、15天、7天）|
| 送检登记 | 记录送检日期、送检机构、送检人、送检费用 |
| 结果录入 | 合格/不合格/需调整 |
| 不合格处理 | 不合格仪器标记"停用"，触发维修或报废流程 |
| 检定完成 | 系统自动计算下次检定日期 |

#### 4.3.3 证书管理（新增功能，MES没有的）
- **上传存档**：检定证书 PDF/图片上传，关联到对应仪器
- **证书到期预警**：检定有效期跟踪
- **历史证书归档**：同一仪器多次检定，历史证书全部保留
- **证书下载**：按需导出打印
- **模板打印**：自检/内校场景，支持生成标准格式证书模板

#### 4.3.4 检定机构管理
- 外部检测院名录（名称、资质、联系方式、服务范围）
- 内部自检/内校管理
- 送检历史记录
- **目前合作检测院**：2家，年度合同约 ¥80万

### 4.4 检定合同管理与对账（最关键的创新功能）

这是对整个系统价值最大化的核心模块，解决人工核对耗费大量时间的问题，**也是后续财务报账的核心数据来源**。

#### 4.4.1 合同管理

##### 合同版本管理（PDF 按年归档）

每年度与各检测院签订的检定服务合同，以"年度"为维度进行管理：

**合同生命周期**
```
原始合同（双方盖章版 PDF 扫描件）
    ↓ 如有补充协议或价格调整
补充协议版（新 PDF，关联原合同）
    ↓ 年底结算
年度结算版（最终确认版）
```

**每个合同记录包含：**
| 项目 | 说明 |
|------|------|
| 合同编号 | 年度+流水号，如 HT-2026-001 |
| 合同名称 | 如"2026年度XX检测院检定服务合同" |
| 检测院 | 关联检定机构 |
| 签订日期 | |
| 合同有效期 | 起止日期 |
| 合同总金额 | 汇总金额 |
| 付款条件 | 预付款/分期/到货后付等 |
| 开票信息 | 开票抬头、税号、开户行等信息 |
| 发票要求 | 增值税专票/普票、税率 |
| 状态 | 待执行/执行中/已完结/已归档 |

**核心：PDF 版本管理**
- 每份合同可上传 **多个版本的 PDF**
- 每个版本记录：版本号、上传时间、上传人、版本说明（如"初次签署版"、"第一次补充协议"、"年底结算版"）
- **最新版设为默认**，历史版本可追溯查看
- 系统自动记录每次上传的时间戳，合同操作日志留痕

##### 合同结构化数据提取（关键）

上传 PDF 后，系统需将合同中的核心信息**结构化存储**，不依赖 PDF 本身（PDF 只作为原始凭证存档）：

**合同明细条目（最重要）**
```
合同编号 HT-2026-001
检测院：XX省计量科学研究院
────────────┬─────────┬──────┬──────┬───────
 仪器名称    │ 型号    │ 数量 │ 单价 │ 小计
────────────┼─────────┼──────┼──────┼───────
 数显卡尺   │ 0-150mm│  30  │ ¥50  │ ¥1,500
 电子天平   │ 200g/0.1│  10  │ ¥80  │ ¥800
 万能试验机 │ 100kN   │  2   │ ¥500 │ ¥1,000
 压力表     │ 1.6级   │  50  │ ¥15  │ ¥750
 温度计     │ -40~100 │  20  │ ¥25  │ ¥500
────────────┴─────────┴──────┴──────┴───────
 合计: 112台       总金额: ¥4,550
```

**录入方式（两种均可）：**
1. **手动录入** — 逐条填写设备名称、数量、单价、小计
2. **Excel 导入** — 从合同附带的报价清单 Excel 直接导入

> ⚠ **不建议自动 OCR 解析 PDF 提取明细**，合同格式不统一，识别错误率高，报账时出问题后果严重。手动录入或 Excel 导入更可靠。

##### 合同数据与检定执行的自动关联

这是后续对账和报账的基础逻辑：

```
合同录入（名称、数量、单价、检测院）
        ↓
实际送检时，选择"关联合同条目"
        ↓
系统自动带入该条目的：设备名称、单价、检测院
        ↓
执行完成录入：实际数量、实际费用、检定结果
        ↓
系统自动生成对比：合同价 vs 实际价
```

**关联方式：**
- 送检登记时，从合同下拉选择对应的**合同条目**
- 也可批量关联：批量送检时，一次选择合同中的所有条目
- 关联后，该次送检的**合同价格自动带入**（不用手填）
- 如果实际费用与合同不同，修改后系统自动标记为"价格差异"

##### 财务报账支持

合同对账的最终目标是**为财务报账提供准确的数据支撑**：

**年度结算清单**
- 按检测院汇总：合同金额 vs 实际发生金额 vs 差异
- 自动生成**结算对账单**，可直接作为报账附件打印

**费用明细导出**
- 按部门导出：该部门全年送检明细（设备、送检时间、费用金额）
- 按检测院导出：该检测院全年服务明细
- 按时间导出：指定时间段的检定费用汇总
- **导出格式**：Excel（财务可直接使用的格式）

**报账辅助功能**
| 功能 | 说明 |
|------|------|
| 合同价参考 | 报账时自动显示合同中对应的单价和总价 |
| 差异预警 | 实际金额与合同金额差异超过阈值（如±5%）时，报账前需确认 |
| 历史价格对比 | 与去年同批次检定的价格对比，辅助判断涨价是否合理 |
| 发票信息关联 | 记录发票号、开票日期、金额，与合同/执行记录关联 |
| 付款状态跟踪 | 已付款/部分付款/未付款，支持多笔付款拆分 |

**报账操作流程（建议）**
```
① 查看对账结果 → 确认无误
② 生成结算对账单（系统自动）
③ 导出 Excel/打印 PDF
④ 作为报账附件提交财务
⑤ 财务付款后，在系统中登记发票号和付款金额
⑥ 状态更新为"已付款"→ 档案归档
```

##### 部门内部检定计划
- 各部门根据年度预算，制定本部门的检定费用计划
- 计划条目：设备、数量、预估费用
- 可与年度合同联动，查看各部门费用消耗进度

#### 4.4.2 实际执行记录

每次检定完成后，系统自动生成或人工录入**实际执行记录**：

| 字段 | 说明 |
|------|------|
| 关联仪器 | 哪台仪器 |
| 送检部门 | 哪个部门送的 |
| 检测机构 | 哪个检测院做的 |
| 送检日期 | |
| 检定结果 | 合格/不合格 |
| 实际数量 | |
| 实际单价 | |
| 实际总金额 | |
| 证书编号 | 关联证书 |

#### 4.4.3 自动对账（核心功能）

系统自动比对**合同明细 ↔ 实际执行记录**，识别差异并高亮显示：

| 对比维度 | 比对内容 | 差异类型 |
|---------|---------|---------|
| **设备名称** | 合同约定的设备 vs 实际送检的设备 | 品种不符、漏检、多检 |
| **数量** | 合同数量 vs 实际数量 | 数量差异（多/少）|
| **单价** | 合同单价 vs 实际单价 | 价格变动 |
| **金额** | 合同金额 vs 实际发生金额 | 总金额差异 |
| **检测机构** | 合同指定机构 vs 实际执行机构 | 机构变更 |
| **部门** | 各部门的合同额度 vs 实际消耗 | 部门费用超支/结余 |

**对账结果展示：**
```
┌──────────────┬────────┬────────┬──────────┐
│  对比项目   │ 合同   │ 实际   │  差异    │
├──────────────┼────────┼────────┼──────────┤
│  检定设备数 │ 120台  │ 118台  │ -2台 🔴  │
│  总金额     │ ¥86,500│ ¥79,200│ -¥7,300🔴│
│  A部门费用  │ ¥32,000│ ¥35,600│ +¥3,600🔴│
│  B部门费用  │ ¥25,000│ ¥23,100│ -¥1,900🟢 │
└──────────────┴────────┴────────┴──────────┘
```

**差异详情可下钻查看**：点击差异数字，展开具体是哪台设备、哪个批次有问题。

#### 4.4.4 差异处理与确认

| 操作 | 说明 |
|------|------|
| 标记为合理差异 | 经确认差异合理（如设备替换），标记后存档 |
| 发起调整 | 需要跟检测院或部门沟通调整的，生成调整单 |
| 导出对账报告 | 一键导出完整的对账结果（Excel/PDF）|
| 历史追溯 | 历年对账记录可查 |

#### 4.4.5 年度检定费用总览

- 各检测院的年度合同执行率（实际金额/合同金额）
- 各部门的检定费用排名
- 合同外临时检定费用统计
- 与往年费用对比分析

| 维修费用统计 | 按部门/时间汇总维修成本 |

### 4.8 测量管理体系监督模块

本模块解决测量管理体系的**日常监督执行**问题，覆盖「部门自查 → 工厂督查 → 问题整改 → 跟踪闭环」全流程。

#### 4.8.1 监督体系概述

```
各部门测量人员
     ↓ 自查填报
部门内部监督（每月/每季度）
     ↓ 部门领导审核
审核通过
     ↓
工艺质量管理科
     ↓ 监督检查
工厂级监督抽查
     ↓ 发放不符合项
各部门整改
     ↓ 整改反馈
工艺质量管理科复核确认
     ↓
监督档案归档
```

#### 4.8.2 部门内部监督执行

**监督模板管理**
- 制定标准化的**监督检查项目清单**（如：仪器台账是否更新、检定是否超期、证书是否齐全、使用记录是否规范等）
- 每个检查项目包含：检查内容、检查标准、评分/判定标准（合格/不合格/不适用）
- 模板可按部门类型定制（不同部门的测量设备不同，检查重点不同）
- 模板版本管理：每次修订记录版本号，历史版本可追溯

**监督计划**
- 各部门按月度/季度制定内部监督计划
- 计划内容：监督时间、检查项目范围、执行人
- 系统自动提醒：计划到期前N天通知部门测量员

**监督填报**
- 部门测量员登录系统，按模板逐项填写监督结果
- 每项结果：合格/不合格/不适用，可附加文字说明
- 不合格项自动触发**纠正措施流程**
- 支持附件上传（现场照片、记录表扫描件等）
- 支持草稿保存，填写完成后提交

**部门内审**
- 部门领导审核本部门的监督填报
- 审核结果：通过 / 退回修改
- 退回需填写修改意见，系统通知测量员修改后重新提交

#### 4.8.3 工厂级监督检查（工艺质量管理科）

这是工艺质量管理科（大哥部门）的核心职责之一，**对其他部门执行情况的监督检查**。

**监督检查计划**
- 工艺质量管理科制定年度/季度**工厂级监督检查计划**
- 按部门轮检：每季度检查N个部门，确保年内覆盖全部部门
- 也可开展专项检查：针对特定问题（如某类仪器超期多、某部门台账混乱等）进行专项督查
- 计划内容：检查部门、检查范围、检查时间、检查组成员

**现场检查执行**
- 检查组成员按检查清单逐项检查：
  - 查看仪器台账是否完整更新
  - 抽查仪器实物与台账是否一致
  - 检查检定证书是否在有效期内
  - 检查仪器使用记录是否规范
  - 检查上次监督发现的问题是否已整改
- 每项结果：合格/不合格/不适用，附**检查人、检查意见**
- 支持现场拍照上传作为证据

**不符合项管理**
- 检查中发现的不符合项，系统自动生成**不符合项报告（NCR）**
- 不符合项内容：
  - 编号、检查部门、检查日期
  - 不符合项描述（什么、为什么不符合）
  - 不符合性质（一般/严重）
  - 关联的检查项目
  - 关联的仪器/记录（如有）
- 不符合项状态流转：
  ```
  已开具 → 已接收 → 整改中 → 整改完成 → 复核确认 → 关闭
  ```

**不符合项整改要求**
- 工艺质量管理科在系统中向被检查部门**下达整改通知**
- 通知内容：整改要求、整改期限、责任人
- 系统自动发送通知（系统内消息/邮件）

**整改跟踪与验证**
- 被检查部门在规定期限内提交**整改措施**和**整改证据**
- 整改措施：问题原因分析、具体整改行动
- 整改证据：整改后的照片/文件/记录
- 工艺质量管理科对整改结果进行**复核验证**
- 复核结果：通过 → 关闭不符合项；不通过 → 退回继续整改
- 超期未整改：系统自动升级提醒（部门领导 → 分管领导）

#### 4.8.4 监督统计与报告

| 报表名称 | 说明 |
|---------|------|
| 部门自查完成率 | 各部门按时完成内部监督的比例 |
| 不符合项统计 | 按部门、按性质、按时段统计不符合项数量和分布 |
| 整改及时率 | 各部门按时完成整改的比例 |
| 超期未整改清单 | 超期尚未关闭的不符合项 |
| 监督检查覆盖率 | 全年工厂级检查覆盖的部门比例 |
| 监督分析报告 | 季度/年度监督情况综合分析与趋势 |

#### 4.8.5 监督档案管理

- 每次监督填报、检查记录、不符合项报告、整改资料**全部电子化存档**
- 按部门、按时间、按监督类型快速检索
- 支持导出全套监督档案（用于体系审核、外部认证等场景）

### 5.1 技术选型（Hermes 自主开发）

由于系统由 Hermes Agent 自主开发并维护，技术选型以 **开发效率高、维护简单、AI 友好** 为原则：

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| 前端 | Vue 3 + Element Plus | 成熟企业级 UI 框架，组件丰富，AI 生成效率高 |

| 文件存储 | 本地文件系统 | 证书 PDF、合同 PDF、附件等存储 |
| 认证 | JWT 登录 | 简单安全，无需额外基础设施 |

## 八、电子签名方案

审批流程要求支持电子签名。考虑到工厂内网环境和使用场景，采用以下方案：

### 8.1 签名方式

**方案：手写签名图片 + 审批人身份验证（推荐）**
- 首次使用时，审批人用手机/手写板手写签名，拍照上传
- 签名图片加密存储，与审批人账号绑定
- 每次审批时，系统自动在审批文书上叠加签名图片
- 审批人需通过登录密码/手机验证码验证身份后方可签名
- 签名记录附带时间戳、IP地址、审批人信息，形成审计链

### 8.2 签名应用场景

| 场景 | 签名形式 |
|------|---------|
| 新购申请审批 | 审批人在电子审批单上签名 |
| 报废申请审批 | 审批人在报废单上签名 |
| 监督审核确认 | 审核人在监督报告上签名 |
| 不符合项关闭确认 | 复核人在不符合项报告上签名 |
| 年度结算对账单 | 体系管理员在对账单上签名（作为报账附件）|

### 8.3 签名文件输出

- 审批完成后，系统生成带有电子签名的 PDF 文件
- PDF 包含：审批内容、签名图片、审批人姓名、时间戳
- 可下载保存，作为原始凭证存档
- PDF 自带文档属性水印（非视觉水印，防止篡改伪造）
